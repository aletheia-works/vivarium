Reproducing numpy/numpy#28287

NumPy's timedelta64 ordering is non-transitive when one of the values uses the generic unit. With x = 1 ms, y = 2 (generic), and z = 5 ns, NumPy reports x < y and y < z but x > z — a transitivity violation in a built-in comparison operator.

Reproduction script

import sys
import numpy as np

x = np.timedelta64(1, "ms")
y = np.timedelta64(2)
z = np.timedelta64(5, "ns")

x_lt_y = bool(x < y)
y_lt_z = bool(y < z)
x_lt_z = bool(x < z)

    "transitivity_violated": x_lt_y and y_lt_z and not x_lt_z,
}
