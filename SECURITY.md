# Security Policy

## Supported versions

Vivarium is in **Phase 0 — Bootstrap**. There are no tagged releases and no
production-facing artefact yet. The only supported branch is `main`.

| Branch | Supported |
| ------ | --------- |
| `main` | ✓         |

This policy is written now so it is in place when the Phase 1 PoC ships
browser-executed code — the first point at which a meaningful attack surface
exists.

## Reporting a vulnerability

### Sensitive reports

For anything that could become a real security issue once Vivarium ships
executable artefacts — potential sandbox escapes, supply-chain concerns,
credential exposure in CI workflows, or similar — please use **GitHub's
[private vulnerability reporting](https://github.com/aletheia-works/vivarium/security/advisories/new)**.

Private reports are visible only to repository maintainers. We will
acknowledge receipt and work with you on the advisory before any public
disclosure.

### Non-sensitive hardening suggestions

Suggestions that are not exploitable in the current codebase — tightening
CI permissions, upgrading a dependency for hygiene, adjusting a workflow
step — belong in a normal
[GitHub Issue](https://github.com/aletheia-works/vivarium/issues/new).

## Response expectations

Vivarium is an open-source project, not a managed service with an SLA. See
[Non-goals: We are not a managed service with an SLA](docs/docs/non-goals.md#we-are-not-a-managed-service-with-an-sla).

Response is **best-effort**:

- We will acknowledge private reports when we see them.
- We do not commit to a fixed triage or patch timeline.
- We will coordinate disclosure timing with you before making an advisory
  public.

## A note on AI-generated reports

Vivarium exists in part _because_ AI-generated bug reports have become a
signal-to-noise problem for open-source maintainers. The cURL precedent is
instructive: by early 2026, fewer than 5 % of security submissions were valid,
with roughly 20 % explicitly AI-generated by the reporter's own admission —
see [vision.md](docs/docs/vision.md) for the full context.

We ask that any security report, AI-assisted or otherwise, include a
human-verified reproduction. A report that cannot be reproduced will be
closed. Vivarium is precisely the tool for building that reproduction before
you file.

## No bug-bounty programme

There is no bug-bounty programme at Phase 0. The project has no budget and
no shipped attack surface that warrants one. The cURL precedent argues against
bounties at this scale: they attract volume, and volume without quality creates
more work than it resolves.
