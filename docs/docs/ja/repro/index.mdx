      各ギャラリーページは <code>&lt;meta name="vivarium-contract" content="v1"&gt;</code>{' '}
      を宣言し、<code>data-verdict="pending" | "pass" | "fail"</code> を持つ{' '}
      <code>#verdict</code> DOM 要素と、<code>window</code> 上の構造化{' '}
      <code>__VIVARIUM_RESULT__</code> エンベロープを公開する。
    </p>
    <Callout>
      "pass" verdict はバグが再現されたことを意味する。"fail" verdict はランタイムが
      修正済みバージョンを出荷したことを意味する。セマンティクスはプロジェクトのドメイン語に
      合わせている: 再現が成功するのは、再現できたときだ。
    </Callout>

    <p>
      各ギャラリーページには CLI バリアントが付属する——<code>repro.py</code>、
      <code>repro.rb</code>、<code>repro.php</code>、または Rust の場合は同じ{' '}
      <code>Cargo.toml</code> + <code>src/main.rs</code>——本物のネイティブインタープリタ/コンパイラに
      対して同じロジックを実行する。リポジトリルートの <code>.mise.toml</code> が、
      WASM ランタイムがバンドルするものと同じ major.minor.patch（または pip 依存ピン）に
      各ランタイムをピン留めする。
    </p>

    <pre><code>{`mise install                                                    # マシンごとに一度
mise exec uv   -- uv run src/layer1_wasm/pandas-56679/repro.py
mise exec uv   -- uv run src/layer1_wasm/numpy-28287/repro.py
mise exec uv   -- uv run src/layer1_wasm/cpython-137205/repro.py
mise exec ruby -- ruby src/layer1_wasm/ruby-21709/repro.rb
mise exec php  -- php  src/layer1_wasm/php-12167/repro.php
mise exec rust -- cargo run --release --manifest-path src/layer1_wasm/regex-779/Cargo.toml`}</code></pre>
