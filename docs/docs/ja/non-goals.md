# 非目標

> Vivarium が構築して*いないもの*、そしてその理由。
>
> 終わりのないプロジェクトにおいて「No」と言うことは構造的に重要な役割を担う。
> 明示的な非目標はスコープを誠実に保ち、正当な拡張をドリフトと区別可能にする。

非目標は「できない」という宣言ではない。「しないことを選んだ」という宣言だ。
以下の各項目には理由が添えられており、将来のコントリビューターが
スコープのドリフトと真の方針転換を見分けられるようにしている。

---

## 汎用コード実行プレイグラウンドではない

StackBlitz、CodeSandbox、Replit、GitHub Codespaces はホスト環境でコードを
書いて実行できるツールとして存在する。Vivarium はそれらと競合しない。
Vivarium はより狭いプリミティブを提供する——特定のバグを再現する——という目的に特化しており、
エディタの人間工学、プロジェクトのスキャフォールド、任意言語の IDE 機能は意図的に持たない。

**なぜか:** 「任意のコードを実行する」と「バグを再現する」を混同するとサーフェス面積が
肥大化する。前者は複数の成熟したプレイヤーによってすでにカバーされており、後者こそが
Vivarium が存在するためのカバーされていないプリミティブだ。

## AI コードジェネレーターではない

Vivarium は修正を書いたり、パッチを提案したり、プルリクエストを生成したりしない。
バグに関する主張が再現可能かどうかを検証する。

**なぜか:** 生成は競争の激しい市場だ（Copilot、Cursor、Claude Code、Codex、
Greptile、Sourcery）。検証は、AI 生成からメンテナーの信頼へのループを閉じる
補完的なプリミティブだ。私たちが検証サイドに留まることで、
誰が・何が元の主張を書いたかに関わらず、出力を信頼できるようになる。

## IDE もエディタでもない

Vivarium にオーサリング UX は存在しない。再現の入力は Issue 本文、PR の diff、
またはリンクされたアーティファクトから来る——私たちがメンテナンスするライブエディタ
ペインからではない。

**なぜか:** エディタはそれ自体が一つのドメインだ。プロジェクトは良い再現基盤を
完成させる前に、エディタ機能の構築に消耗することになる。

## CVE トリアージでも脆弱性スキャナでもない

GitHub Security Lab、Snyk、Semgrep その他は脆弱性を分類し、アドバイザリを
維持管理する。Vivarium は重大度を割り当てたり、アドバイザリを発行したり、
CVE データベースを管理したりしない。「これを再現できるか？」という問いに答えるだけだ——
セキュリティワークフローがその結果を使うかもしれないが、Vivarium 自体はセキュリティ
プロダクトではない。

**なぜか:** セキュリティの分類には、私たちのスコープ外にある専門家の判断と
規制への整合が必要だ。両者を混ぜると双方が薄まる。

## SLA を持つマネージドサービスではない

Vivarium は OSS ファーストだ。ホスト型の付帯サービスは将来存在するかもしれないが、
プロダクトはあくまでオープンソースプラットフォームだ。ホスト型ティアは付帯サービスであって、
プロダクトとしての本契約ではない。

**なぜか:** 私たちが大切にするプリミティブ——再現——は、オープンプロジェクトの
メンテナーが許可を求めることなく自分のインフラで実行できて初めて目的を果たす。
マネージドティアを維持するためにソースをクローズすることは、
プロジェクトが存在する理由を損なう。

## 単一の技術に縛られていない

WebAssembly はプロダクトではない。Docker はプロダクトではない。
Record-replay はプロダクトではない。プロダクトは再現であり、
技術は問題によって選ばれる。

**なぜか:** 隣接するすべてのプレイヤーは単一の技術に縛られている
（StackBlitz は WASM、Codespaces は Docker、`rr` は Linux ptrace）。
技術の境界がプロダクトの境界になり、到達できないカテゴリは永続的に到達不能になる。
Vivarium はそのような固定を拒否する。

## 3 年の MVP ではない

ロードマップは四半期ではなく年単位で広がる。プロジェクトは終わりのないスコープには
終わりのない忍耐が必要だという前提の上に構築されている:
ローンチへの出荷圧力なし、資金調達を正当化するための機能クォータなし、
完了期限なし。

**なぜか:** 私たちが尊重する技法——record-replay 研究、決定論的シミュレーション、
成熟した WASM ランタイム——は成熟するのに数十年かかった。
プリミティブとしての再現も同じタイムスケールに値する。
