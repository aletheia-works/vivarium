  <Section
    eyebrow="// CONTRACT"
    heading="Every page satisfies vivarium-contract: v1."
  >
    <p>
      Each gallery page declares <code>&lt;meta name="vivarium-contract" content="v1"&gt;</code>{' '}
      and exposes a <code>#verdict</code> DOM element with{' '}
      <code>data-verdict="pending" | "pass" | "fail"</code>, plus a structured{' '}
      <code>__VIVARIUM_RESULT__</code> envelope on <code>window</code>.
    </p>
    <Callout>
      A "pass" verdict means the bug reproduced. A "fail" verdict means the
      runtime ships a fixed version. The semantics match the project's domain
      noun: a reproduction succeeds when it reproduces.
    </Callout>
  </Section>

  <Section
    eyebrow="// NATIVE RE-VERIFICATION"
    heading="Same logic against a real interpreter."
  >
    <p>
      Each gallery page has a companion CLI variant — <code>repro.py</code>,{' '}
      <code>repro.rb</code>, <code>repro.php</code>, or (for Rust) the same{' '}
      <code>Cargo.toml</code> + <code>src/main.rs</code> — that runs the same
      logic against a real native interpreter / compiler. The{' '}
      <code>.mise.toml</code> at the repo root pins each runtime to the same
      major.minor.patch (or pip dependency pin) the WASM runtime bundles.
    </p>

    <pre><code>{`mise install                                                    # one-time per machine
mise exec uv   -- uv run src/layer1_wasm/pandas-56679/repro.py
mise exec uv   -- uv run src/layer1_wasm/numpy-28287/repro.py
mise exec uv   -- uv run src/layer1_wasm/cpython-137205/repro.py
mise exec ruby -- ruby src/layer1_wasm/ruby-21709/repro.rb
mise exec php  -- php  src/layer1_wasm/php-12167/repro.php
mise exec rust -- cargo run --release --manifest-path src/layer1_wasm/regex-779/Cargo.toml`}</code></pre>
